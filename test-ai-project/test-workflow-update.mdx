## Recent updates

### Test suite and dev container setup

The project now includes a test suite and a dev container configuration for streamlined development:

- **Test suite**: added `test_sequential_tagging.py`, `test_sequential_minimal.py`, `test_simplified_relationships.py`, `test_l1_l6_simple.py`, and `test_app_startup.py` covering sequential path generation, relationship extraction, and app startup filtering logic
- **Dev container**: added `.devcontainer/devcontainer.json` with Python 3.11, automatic dependency installation, and Streamlit auto-launch on port 8501
- **Speed benchmarking**: added `speed_test.py` for profiling tagging performance across batch sizes

### Sequential tagging with relationship preservation

The knowledge tagging system now uses sequential learning paths instead of flat L1-L6 category arrays. This change was introduced in the `feature/l1-l6-simple` branch.

- **New `sequential_paths` format**: each path is an ordered array representing a coherent learning progression from L3 through L6 (e.g., `["Memory", "Long-term Memory", "Persistence", "Vector Storage"]`)
- **Automatic relationship extraction**: the system derives L3-to-L4, L4-to-L5, and L5-to-L6 mappings directly from path structure, guaranteeing mathematical correctness
- **~90% speed recovery**: processing performance is nearly back to baseline after the restructuring
- **Backward compatible**: existing L3-L6 flat arrays and the Streamlit UI continue to work without changes
- **Data storage updated**: `data/links_store_v2.csv` now includes `sequential_paths` and `knowledge_paths` columns
